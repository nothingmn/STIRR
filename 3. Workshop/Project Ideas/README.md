# Project Ideas

Future projects not yet started — the backlog of possibilities awaiting activation. This area holds project concepts, proposals, and aspirations that have not yet moved to active development. It is the Workshop's waiting room: where ideas are captured, elaborated, and prioritized before committing resources.

## What belongs here

- Project concept notes and pitch documents
- Feasibility sketches and initial scope outlines
- Priority rankings and backlog lists
- "Someday/maybe" items with enough substance to preserve
- Dependencies and prerequisites for future projects

## What does NOT belong here

- Active projects → [[../Prototyping/README.md|Prototyping]] or a dedicated project space once started
- Half-formed ideas without project shape → [[../../1. Study/Thought Streams/README.md|Thought Streams]]
- Quick experiments already underway → [[../Prototyping/README.md|Prototyping]]
- Completed project records → [[../../7. Archive/Completed Projects/README.md|Completed Projects]]

## Tips

- **Estimate effort.** Every project idea should include a rough effort estimate (small/medium/large or hours) and a priority tag. This enables backlog triage.
- **Name descriptively.** Title project ideas by their intended outcome, not by a code or number. "CLI tool for X" beats "Project 7."
- **Revisit regularly.** The ideas backlog should be reviewed periodically. Promote, deprioritize, or discard. Do not let it become a graveyard.
- **Link dependencies.** If one idea depends on another being completed first, link them. Dependency chains help during review.
