# Obsolete

Content kept for the record but no longer active — the vault's cold storage.

## What belongs here

- Superseded documentation and outdated reference material
- Deprecated templates and configurations
- Previous versions of notes replaced by newer ones
- Abandoned approaches or disproved hypotheses
- Any content no longer active but too significant to delete

## What does NOT belong here

- Active content of any kind (it belongs in its proper room)
- Content that could still drive action (if it's actionable, it's not obsolete)
- Duplicate files with no unique content (delete, do not archive)
- Sensitive data that should be securely deleted rather than retained

## Area-specific rules

1. **Mark as obsolete.** Every note here should carry an `obsolete: true` frontmatter property and, where possible, a note explaining what superseded it with a link to the replacement.
2. **No maintenance.** Obsolete content is not updated. If information changes, the replacement note is updated — not this one.
3. **Link to replacement.** When content is moved here because something else replaced it, link to the replacement. The chain from obsolete to current should be traceable.
