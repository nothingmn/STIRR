# Obsolete

Content kept for the record but no longer active — the vault's cold storage. This area holds content that has been superseded, debunked, or otherwise rendered inactive but is retained because discarding it would lose information or break references. Obsolete content is archived, not maintained.

## What belongs here

- Superseded documentation and outdated reference material
- Deprecated templates and configurations
- Previous versions of notes replaced by newer ones
- Abandoned approaches or disproved hypotheses
- Any content no longer active but too significant to delete

## What does NOT belong here

- Active content of any kind → it belongs in its proper room
- Content that could still drive action → if it's actionable, it's not obsolete
- Duplicate files with no unique content → delete, do not archive
- Sensitive data that should be securely deleted rather than retained

## Tips

- **Mark as obsolete.** Every note here should carry an `obsolete: true` frontmatter property and, where possible, a note explaining what superseded it with a link to the replacement.
- **No maintenance.** Obsolete content is not updated. If information changes, the replacement note is updated — not this one.
- **Link to replacement.** When content is moved here because something else replaced it, link to the replacement. The chain from obsolete to current should be traceable.
- **Retain, don't reference.** Do not link to obsolete content from active notes unless making an explicit historical comparison. Obsolete notes should not appear in active workflows.
- **Periodic purges.** Review obsolete content periodically. Material superseded for more than a reasonable retention period (e.g., 2 years) and carrying no legal or historical significance may be deleted.
