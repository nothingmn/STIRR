# Completed Projects

Finished project records — the closed body of work preserved for reference and reuse. This area holds records of projects that have been completed: final reports, deliverables, post-mortems, and all material worth retaining from work that has formally concluded. Completed projects are stored, not maintained.

## What belongs here

- Final project reports and deliverables
- Post-mortem and retrospective notes
- Project archives (requirements, design docs, correspondence)
- Lessons learned and reusable artifacts
- Project metadata (timeline, team, scope summary)

## What does NOT belong here

- Active projects → stay in [[../../3. Workshop/README.md|Workshop]] or their dedicated space
- Project ideas not yet started → [[../../3. Workshop/Project Ideas/README.md|Project Ideas]]
- Historical records not tied to a specific project → [[../Historical/README.md|Historical]]
- Obsolete content with no remaining reference value → [[../Obsolete/README.md|Obsolete]]

## Tips

- **Close before archiving.** Only move a project here once it has a clear closure event (delivery, launch, formal cancellation). Do not archive projects that are merely dormant.
- **Preserve, don't maintain.** Completed project notes are read-only. If a completed project spawns new work, that new work starts as a new project elsewhere.
- **Include a summary.** Every completed project must have a brief summary note: what was done, when, by whom, and the outcome.
- **Index for retrieval.** Tag by technology, domain, client, and outcome type to enable cross-referencing and future reuse.
- **Link to active work.** If a completed project is relevant to current work, link from the active project to the completed archive. Reuse is the purpose of retention.
