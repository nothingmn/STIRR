# Completed Projects

Finished project records — the closed body of work preserved for reference and reuse.

## What belongs here

- Final project reports and deliverables
- Post-mortem and retrospective notes
- Project archives (requirements, design docs, correspondence)
- Lessons learned and reusable artifacts
- Project metadata (timeline, team, scope summary)

## What does NOT belong here

- Active projects (those stay in Workshop or their dedicated space)
- Project ideas not yet started (those stay in [[../../3. Workshop/Project Ideas/PROTOCOL|Project Ideas Protocol]])
- Historical records not tied to a specific project (those go to [[../Historical/PROTOCOL|Historical Protocol]])
- Obsolete content with no remaining reference value (those go to [[../Obsolete/PROTOCOL|Obsolete Protocol]])

## Area-specific rules

1. **Preserve, don't maintain.** Completed project notes are read-only. If a completed project spawns new work, that new work starts as a new project elsewhere.
2. **Index for retrieval.** The LLM actively indexes completed projects for future reference. Tag by technology, domain, client, and outcome type to enable cross-referencing.
