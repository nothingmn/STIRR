# Calendar

Time-bound items, deadlines, and events — anything anchored to a specific date or window. This area holds material that is governed by time: appointments, deadlines, recurring events, and planning horizons. If something must happen by or on a certain date, it belongs here.

## What belongs here

- Deadlines with specific due dates
- Scheduled meetings, appointments, and events
- Recurring reminders and seasonal tasks
- Planning views (weekly, monthly, quarterly outlooks)
- Time-sensitive opportunities with expiry dates

## What does NOT belong here

- Daily reflections or journal entries → [[../Daily/README.md|Daily]]
- Unsorted quick captures → [[../Inbox/README.md|Inbox]]
- Project tasks without hard deadlines → [[../../3. Workshop/README.md|Workshop]] or the relevant project room
- Historical records of past events → [[../../7. Archive/Historical/README.md|Archive/Historical]] once closed

## Tips

- **Every item needs a date.** If there is no time anchor, the item does not belong here. Assign a date or move it elsewhere.
- **Name by date.** Use `YYYY-MM-DD` prefixes or inline date properties so that sorting by filename yields chronological order.
- **Recurring items.** Use Templater or periodic note plugins for recurring events. Do not manually duplicate each occurrence unless each one carries unique context.
- **Expire and archive.** Past events that no longer drive action should be archived. Do not let the Calendar clutter with historical residue.
