# General Reference

Topics not yet warranting their own wing — the stable, encyclopedic knowledge base. This area holds reference material on topics that are useful but not central enough to merit a dedicated room. It is the Library's catch-all: well-organized, precisely indexed, and maintained for retrieval rather than active engagement.

## What belongs here

- Encyclopedic entries on topics, concepts, and definitions
- How-to guides and technical references
- Glossaries, terminologies, and cheat sheets
- Stable knowledge unlikely to change frequently
- Topics being indexed for potential future wing creation

## What does NOT belong here

- Active research notes → [[../../1. Study/Research/README.md|Research]]
- Source records tied to specific bibliographic entries → [[../../1. Study/Sources/README.md|Sources]]
- Project-specific reference → lives with the project in [[../../3. Workshop/README.md|Workshop]]
- Obsolete or superseded material → [[../../7. Archive/Obsolete/README.md|Obsolete]]
- Personal notes → [[../../5. Quarters/README.md|Quarters]]

## Tips

- **Neutral tone.** Reference notes are descriptive, not argumentative. Save analysis for the Study.
- **Consistent structure.** Use a template for reference entries (see [[../../8. Foundation/Templates/README.md|Templates]]). Every note should have the same section structure for reliable retrieval.
- **Promote when mature.** When a reference topic accumulates enough material, it may warrant its own wing. Raise the issue; do not create wings unilaterally.
- **No orphans.** Every reference note should link to at least one other note — either another reference entry, a research note, or a project note. Isolated notes are flags for re-evaluation.
