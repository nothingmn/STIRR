# The Library

Knowledge formalized and structured for retrieval. The explicit layer. Put things here when you need to find them again with high specificity -- every note should have a clear title, location, and relationship to neighbors. The Library does the Karpathy wiki thing: the Indexer maintains the explicit layer so you do not have to.

## How this room works

PRECISION mode, LLM-augmented substrate. This is where the LLM adds the most value with the least risk. Library content is already explicit -- the Conjugate Pair trade-off has already been paid. The Indexer maintains MoCs, cross-references notes, generates reference entries, tags and categorizes material, and identifies gaps. It does not create new analytical frameworks (the Library indexes; the Study thinks) or override your classification decisions.

## What you do here

- **Retrieve from the Library.** Do not do your primary thinking here. The Library is for retrieval, not exploration.
- **When you need a fact, walk to the Library.** The Indexer maintains it.
- **If a Library note feels like it contains developing, holistic thinking,** it may belong in the Study instead.
- **Domain wings are created organically.** Start in General Reference and promote when a topic accumulates enough notes to stand on its own.

## What moves out

- Notes that contain developing or holistic thinking may migrate back to the Study
- Clippings or reference material not actively being used may migrate to Archive/Clippings
- Completed research that originated here and is fully processed stays (the Library is a permanent home for reference material)
